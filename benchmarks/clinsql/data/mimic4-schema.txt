# Concise notes — only commonly confusing columns

- **hadm_id vs stay_id**: `hadm_id` = hospital admission id; `stay_id` = ICU stay id (sub-episode within an admission).
- **charttime vs storetime**: `charttime` = when the event/measurement occurred; `storetime` = when it was recorded/entered/verified. Prefer `charttime` for temporal analyses.
- **itemid**: numeric code for labs/measurements/drug components — lookup in `d_labitems` or `d_items`.
- **value / valuenum / valueuom**: `value` may be text; `valuenum` is the numeric form (use for calculations); `valueuom` is the unit.
- **seq_num**: ordering for billing entries (diagnoses/procedures); lower often indicates higher priority.
- **poe_id / poe_seq**: top-level provider order (`poe_id`) plus its sequence; see `poe_detail` for attribute/value rows.
- **orderid / linkorderid (ICU inputs)**: group related infusion/order records and link rate-change segments over time.
- **interpretation (microbiology)**: susceptibility call (e.g., S/I/R).
